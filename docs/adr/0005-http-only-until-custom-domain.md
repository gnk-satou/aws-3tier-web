# 5. ALB は当面 HTTP のみとし、HTTPS 化はカスタムドメイン取得後に行う

Date: 2026-07-04

## Status

Accepted

## Context

Step 2 で ALB を導入するにあたり、HTTPS(ACM 証明書 + 443 リスナー)を最初から構成するか検討した。

- ACM のパブリック証明書は独自ドメインの所有検証が必須であり、ALB の自動生成 DNS 名(`*.elb.amazonaws.com`)には発行できない
- 本プロジェクトは apply → 検証 → destroy の短期サイクル運用であり、Route 53 でのドメイン維持は月額固定費が発生する
- 自己署名証明書での HTTPS はブラウザ警告が出るため、ポートフォリオの証跡としてむしろ品質を下げる

## Decision

- Step 2 の ALB リスナーは HTTP(80)のみとする
- HTTPS 化(ACM + 443 リスナー + HTTP→HTTPS リダイレクト)は、カスタムドメインを取得した時点で追加ステップとして実施する
- この制約は README に将来課題として明記する

## Consequences

- 通信は平文だが、デモ用静的ページのみで機密データは扱わないため許容する
- HTTPS 化の手順(ACM 検証、リスナー追加、SG 変更不要である点)を将来の学習項目として残せる
